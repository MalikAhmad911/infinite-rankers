## Task 2 — Strengthen About Page + Entity SEO Signals

**Objective**
Improve homepage/entity clarity and strengthen trust/Organization signals using only information already supported by the site/codebase/publicly stated business details.

The audit found weak entity clarity and trust signals. The site needs a clearer business identity, better About-page substance, and proper Organization schema — but everything must remain evidence-based.

**Scope**
- Improve the About page with stronger company-level positioning and business story
- Add a sharper one-line positioning statement on the homepage
- Add/expand Organization schema
- Add `sameAs` only if real profile URLs already exist in the code/content
- Keep everything strictly evidence-based

**Do not**
- Invent founder bio details
- Invent team members
- Invent awards, credentials, or social profiles
- Add unsupported claims

**Content rules**
If no strong founder data exists in the codebase/content, keep the page company-level only.

Allowed only if already supported in the project/public content:
- Infinite Rankers LLC
- Delaware address
- Google Partner
- any real social/profile URLs already present

If those facts are not clearly supported in code/content, do not force them in.

**Implementation guidance**

### 1) Homepage clarity
Add/refine a concise one-line positioning statement near the main hero/intro that clearly says what the company does and for whom.

Goal:
- clearer category definition
- stronger first-impression relevance
- better trust and topical clarity

Do not make it fluffy. Keep it direct and commercially clear.

### 2) About page improvement

### 5) Trust signal cleanup
Where relevant, improve visible trust signals on homepage/About page using supported facts only:

**Suggested files to inspect**

**Acceptance criteria**
- Homepage has a clearer one-line positioning statement
