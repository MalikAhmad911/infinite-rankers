## YOUR AUDIT CHECKLIST

### SECTION 1 — CRAWLABILITY & INDEXABILITY
- Fetch and read robots.txt — is anything blocked that should not be?
- Fetch sitemap.xml — does it exist, load correctly, and list all pages?
- Check every URL in sitemap — does it return 200 status?
- Check for noindex tags on pages that should be indexed
- Check canonical tags — are any pages self-canonicalizing incorrectly?
- Check for redirect chains (301 → 301 → 200)
- Check for soft 404s (page returns 200 but shows error content)
- Check hreflang if applicable
- Check crawl depth — how many clicks from homepage to deepest page?
Report: Full crawlability health map

### SECTION 2 — TECHNICAL SEO
For EVERY page check:
- Title tag: present / unique / under 60 chars / contains primary keyword / compelling
- Meta description: present / unique / under 160 chars / has CTA
- H1: exactly one / contains keyword / not same as title
- H2s: logical hierarchy / keyword variations / not keyword stuffed
- H3s: supporting content / FAQ questions
- Canonical: correct URL / no trailing slash conflicts
- Open Graph: og:title / og:description / og:image / og:url
- Twitter Card tags
- Viewport meta tag
- Charset declaration
- Structured data (JSON-LD)
Report: Tag audit table for every page

### SECTION 3 — SSR / RENDERING AUDIT
- Is content in raw HTML response or only after JavaScript executes?
- Check each page — does Googlebot see the content?
- Are dynamic content sections server rendered?
- Check Time to First Byte (TTFB)
- Check if React hydration is causing content flash
- Check if any important content is behind lazy load
- Verify meta tags are in initial HTML — not injected by JavaScript after load
Report: Rendering status per page type

### SECTION 4 — CONTENT QUALITY AUDIT
For every landing page:
- Word count (flag anything under 800 words)
- Unique content percentage vs other pages on site
- Is H1 unique across all pages?
- Is intro paragraph unique or templated?
- Are FAQs unique or copy-paste with keyword swap?
- Does content read naturally or is it obviously AI generated filler?
- Does content actually help the user or just target keywords?
- Are there real statistics, examples, case studies?
- E-E-A-T signals: Experience, Expertise, Authority, Trust present?
- Author information present?
Report: Content quality score per page (1-10)

### SECTION 5 — SCHEMA MARKUP AUDIT
Check every page for:
- Schema type appropriate for page (LocalBusiness / Service / FAQPage / Article / BreadcrumbList)
- JSON-LD syntax valid — no errors
- Required properties present per schema type
- FAQPage schema matches visible FAQ content exactly
- No conflicting schemas on same page
- BreadcrumbList on all inner pages
- Organization schema on homepage
- WebSite schema with SearchAction on homepage
- SiteLinks SearchBox eligible?
Report: Schema validation per page

### SECTION 6 — DUPLICATE & CANNIBALIZATION AUDIT
- Compare all page titles — any duplicates?
- Compare all H1s — any duplicates?
- Compare all meta descriptions — any duplicates?
- Run content similarity check across all pages — flag over 30% similarity
- Check keyword targeting — are two pages targeting same keyword?
- Check URL variants — /podium-alternative vs /podium-alternatives vs /alternatives-to-podium
- Identify pages that will compete with each other for same query
- Check if blog posts cannibalize landing pages
Report: Full cannibalization risk map

### SECTION 7 — INTERNAL LINKING AUDIT
- Map all internal links across entire site
- Identify orphan pages (zero internal links pointing to them)
- Check anchor text diversity — too many exact match anchors?
- Check if most important pages get most internal links
- Verify footer links are not the only internal links to some pages
- Check link depth — no important page more than 3 clicks from homepage
- Check for broken internal links (404s)
Report: Internal link authority flow map

### SECTION 8 — PAGE SPEED & CORE WEB VITALS
Check each page template:
- Largest Contentful Paint (LCP) — target under 2.5s
- First Input Delay (FID) / Interaction to Next Paint (INP) — target under 200ms
- Cumulative Layout Shift (CLS) — target under 0.1
- Time to First Byte (TTFB) — target under 600ms
- Check image optimization — WebP format / lazy loading / proper sizing
- Check JavaScript bundle size — anything over 200KB is suspicious
- Check CSS — any render-blocking stylesheets?
- Check font loading — font-display: swap used?
- Check third party scripts — any blocking page load?
Report: CWV scores per page template

### SECTION 9 — AIO / GEO / AEO AUDIT
AIO (AI Overview Optimization):
- Does content directly answer the target query in first 100 words?
- Are there clear definition statements?
- Are lists and structured answers present?
- Is content concise enough to be extracted by AI?
- Does page answer related questions beyond the primary keyword?

GEO (Generative Engine Optimization):
- Is Infinite Rankers clearly identified as a real entity?
- Are NAP details (Name, Address, Phone) consistent across all pages?
- Is business category and service area clearly defined?
- Are there clear expertise signals for AI to cite?
- Is content structured so AI can summarize it accurately?

AEO (Answer Engine Optimization):
- Are there question-based headings (H2/H3 starting with How, What, Why, When)?
- Are answers under those questions concise (under 50 words for featured snippet)?
- Does FAQ schema match visible FAQ content?
- Is content formatted for People Also Ask boxes?
- Are there step-by-step processes marked up correctly?
Report: AIO/GEO/AEO readiness score per page

### SECTION 10 — THIN PAGE & TEMPLATE DETECTION
- Flag every page under 800 words
- Flag every page where intro paragraph matches another page over 50%
- Flag pages where only the keyword name changed but structure is identical
- Flag FAQ sections where questions are same across pages with keyword swap
- Flag CTA sections that are 100% identical across all pages (minor variation needed)
- Flag pages with no unique value proposition
- Flag pages Google would likely classify as doorway pages
Report: Thin/Template risk level per page (Critical/High/Medium/Low/Pass)

### SECTION 11 — SITEMAP & ROBOTS AUDIT
- robots.txt: correct directives / no important pages blocked / sitemap referenced
- sitemap.xml: all pages present / correct URLs / lastmod present / priority set / changefreq set
- Check sitemap URL matches actual canonical URL exactly
- Check for pages in sitemap that return non-200 status
- Check sitemap file size under 50MB
- Check sitemap URL count under 50,000
- Is sitemap submitted to Google Search Console? (check if referenced)
Report: Sitemap health full report

### SECTION 12 — URL STRUCTURE AUDIT
- Are URLs clean and descriptive?
- No unnecessary parameters in URLs?
- No uppercase letters in URLs?
- Consistent trailing slash policy?
- URL length reasonable (under 75 chars)?
- Keywords in URL without stuffing?
- Logical folder hierarchy?
Report: URL quality score per page
